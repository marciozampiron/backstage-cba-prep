How do I configure TechDocs publisher and storage settings in Backstage?

psychology

To configure TechDocs publisher and storage settings, you typically modify the app-config.yaml file. The core settings involve defining the publisher type (e.g., local, awsS3, googleGcs, azureBlobStorage) and its specific credentials.

For a local setup, you would use:

techdocs:
  publisher:
    type: 'local'

For cloud storage, you must configure the respective provider credentials securely, often referencing environment variables rather than hardcoding secrets.

Can you explain Software Catalog entity metadata?

psychology

Software Catalog entity metadata in Backstage is defined in YAML files (often named catalog-info.yaml). Every entity must have a core set of metadata that identifies it within the catalog.

The most critical fields are:

  • name: The unique identifier for the entity (e.g., my-backend-service).
  • namespace: (Optional) Used to group related entities or avoid naming collisions. Defaults to default.
  • description: A human-readable summary of the entity's purpose.
  • annotations: Key-value pairs used for integrating with plugins (e.g., linking a GitHub repository or PagerDuty service).
  • tags: A list of strings used for filtering and categorizing in the UI.
